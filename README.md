# 🚧 RoadWatch

**RoadWatch is a fully deployed civic infrastructure platform for road monitoring, complaint management, and transparency in public road maintenance.**

It enables citizens to view real-time road conditions, report issues, and track resolution through a structured accountability system supported by administrative oversight and verified government data sources.

---

## 📌 Overview

RoadWatch centralizes road infrastructure data and citizen complaints into a single transparent platform.

Users can:

* Explore a live **color-coded map of 50+ roads, bridges, and link roads in Mumbai**
* File complaints instantly (manual or AI-assisted)
* Track complaint resolution in real time
* Access verified road-level information through an AI assistant

---

## ❗ Problem Statement

Road infrastructure management today suffers from:

* Fragmented and inconsistent data sources
* Limited transparency in maintenance spending
* Lack of accountability in complaint resolution
* Inefficient grievance reporting systems

---

## 💡 Solution

RoadWatch bridges these gaps by providing a unified, data-driven platform that connects citizens, authorities, and infrastructure information in real time.

It ensures:

* Transparent complaint tracking
* Structured road-level data access
* AI-assisted reporting and query system
* Administrative accountability with privacy preservation

---

## 🚀 Key Features

### 🗺️ Interactive Road Map (Mumbai)

* Live map of **50+ roads, bridges, and link roads**
* Color-coded based on complaint volume:

  * 🔴 High complaints
  * 🟠 Medium complaints
  * 🟢 Low complaints
* Click any road to view detailed information

---

### 📢 Smart Complaint System

Users can submit complaints in two ways:

**1. Manual Complaint**

* Select the road and describe the issue

**2. AI-Assisted Complaint**

* Provide:

  * Road name
  * Issue type
  * Severity
* AI automatically maps the precise location

---

### 🎟️ Ticketing & Email Notification System (Core Feature)

* Every complaint generates a **unique ticket ID**
* Ticket ID is **automatically sent to the user’s email**
* Applies to both manual and AI-generated complaints
* Users receive updates linked to the same ticket throughout the lifecycle

This ensures:

* Reliable complaint tracking
* Easy reference for follow-ups
* Structured accountability across the system

---

### 📊 User Dashboard

Each user has a personal dashboard featuring:

* All submitted complaints
* Real-time status updates
* Visual analytics (graphs and pie charts)
* Detailed complaint timelines:

  * Submitted → Assigned → In Progress → Resolved
* Ticket-based tracking linked directly to email notifications

---

### 🏛️ Admin Dashboard

* Centralized view of all complaints
* No user identity is visible (privacy-preserving design)
* Ticket ID–based management only
* Status control:

  * Assigned
  * In Progress
  * Resolved
* Real-time sync with user dashboards
* Full timestamped lifecycle tracking for every complaint

---

### 🤖 AI Road Assistant

Users can query verified infrastructure data, such as:

* Road type (NH/SH/MDR/local)
* Contractor details
* Budget history (sanctioned and spent)
* Latest repair dates
* Authority 
* Official grievance portal links
* Source references for all information

All data is sourced strictly from **verified government websites and trusted public infrastructure datasets**.

---

## 🔄 System Workflow

1. User explores interactive road map
2. Selects a road and views detailed information
3. Submits complaint (manual or AI-assisted)
4. System generates **unique ticket ID**
5. Ticket ID is **sent to user via email**
6. Complaint is routed to the admin dashboard
7. Admin updates status in real time
8. User tracks full lifecycle via dashboard timeline
